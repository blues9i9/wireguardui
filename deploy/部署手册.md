# WireGuard UI 部署手册

## 环境信息

| 项目 | 值 |
|------|-----|
| 服务器 IP | 192.168.18.128 |
| 操作系统 | Rocky Linux 9.8 |
| 用户名 | root |
| WireGuard UI 版本 | v0.6.2 (中文版) |
| 监听地址 | `0.0.0.0:5000` |
| Web 管理地址 | `http://192.168.18.128:5000` |
| 默认账号 | admin / admin |

---

## 1. 安装依赖

```bash
dnf install -y go wireguard-tools
```

## 2. 创建目录

```bash
mkdir -p /opt/wireguard-ui/db
mkdir -p /etc/wireguard
```

## 3. 上传二进制

从本地上传编译好的二进制文件：

```bash
scp wireguard-ui-linux root@192.168.18.128:/opt/wireguard-ui/wireguard-ui
```

> 如果要在服务器上直接编译，将整个项目源码的 `main.go`、`go.mod`、`go.sum` 及 `handler/` 等目录复制到服务器，在 `/opt/wireguard-ui/` 目录下执行 `go build -o wireguard-ui main.go`。

## 4. 设置权限

```bash
chmod +x /opt/wireguard-ui/wireguard-ui
```

## 5. 生成 Session Secret

```bash
openssl rand -hex 32
```

## 6. 创建 systemd 服务

创建 `/etc/systemd/system/wireguard-ui.service`：

```ini
[Unit]
Description=WireGuard UI
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/wireguard-ui
ExecStart=/opt/wireguard-ui/wireguard-ui
Restart=on-failure
RestartSec=5
Environment=WGUI_USERNAME=admin
Environment=WGUI_PASSWORD=admin
Environment=WGUI_SESSION_SECRET=<替换为第5步生成的密钥>

[Install]
WantedBy=multi-user.target
```

## 7. 启动服务

```bash
systemctl daemon-reload
systemctl enable wireguard-ui
systemctl start wireguard-ui
systemctl status wireguard-ui
```

## 8. 配置防火墙

```bash
firewall-cmd --permanent --add-port=5000/tcp    # Web UI
firewall-cmd --permanent --add-port=51820/udp   # WireGuard VPN
firewall-cmd --reload
```

## 9. 验证部署

```bash
# 检查 HTTP 响应码（应为 307 重定向到登录页）
curl -s -o /dev/null -w "%{http_code}" http://localhost:5000/

# 或直接查看登录页
curl -s http://localhost:5000/login
```

浏览器访问 `http://192.168.18.128:5000`，使用 `admin / admin` 登录。

## 环境变量说明

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `BIND_ADDRESS` | 监听地址和端口 | `0.0.0.0:5000` |
| `WGUI_USERNAME` | Web 登录用户名 | `admin` |
| `WGUI_PASSWORD` | Web 登录密码 | `admin` |
| `SESSION_SECRET` | Session 加密密钥（务必设置随机值） | 自动生成 |
| `WGUI_CONFIG_FILE_PATH` | WireGuard 配置文件路径 | `/etc/wireguard/wg0.conf` |
| `DISABLE_LOGIN` | 禁用登录认证（不安全） | `false` |

完整变量列表见项目 README 或执行 `./wireguard-ui --help`。

---

## 安全建议

1. 修改默认密码：登录后在 **用户设置** 页面修改 admin 密码
2. 为 `SESSION_SECRET` 设置足够复杂的随机值
3. 如有公网暴露，建议配置 Nginx 反向代理 + HTTPS
4. 建议创建独立运行用户而非直接使用 root
